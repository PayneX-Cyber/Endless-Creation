# 小说创作 3a.1：灵感模式入口与对话式创意收集 切片规格

日期：2026-07-02
上位文档：`docs/plans/2026-06-30-novel-creation-migration-plan.md`、`docs/plans/2026-07-02-novel-generation-chain-3a-spec.md`

## 定位

3a 的「弹窗三步生成」链路功能成立但入口太工具化，不符合 NovelForge「从一个故事想法进入 outline / blueprint，再保存成小说项目」的创意模式定位。

3a.1 **不推翻 3a 代码**，只给它换一个前门：入口流程重构 + 对话式创意收集，收集结果沉淀为 `idea` 后接入现有 3a 蓝图/大纲链。

说明：灵感模式在 6-30 文档中原属后置参考能力（第一阶段明确禁止展示）。现将其最小形态提前为 3a.1，是有意决策——理由是它只消费已证明的 3a 链，数据层零改动，成本集中在 UI 与一个对话 prompt。

## 前置门槛

**3a 本体验收（build + 8 条 QA）通过后，3a.1 方可开工。** 不得在未验证的 3a 链上叠加入口层。

> 2026-07-02 更新：门槛已通过——build 全绿，8 条 QA 代码级审计逐条通过（详见验证记录）。3a.1 放行。

## 流程范围

```text
创作中心（小说创作页新首页）
  ├── 卡片 1：灵感模式 → 启动页 → 对话式灵感空间 → 沉淀为创意 → 现有 3a 蓝图/大纲向导
  └── 卡片 2：小说工作台 → 现有三栏工作台（不变）
```

### 1. 创作中心

- 小说创作页首页改为两张卡片：「灵感模式」「小说工作台」。
- 风格参考 NovelForge 创作中心截图，**适配 Endless Creation 蓝色桌面端主题**，不照搬配色。
- 只做两张卡片，不加第三张占位卡。

### 2. 灵感模式启动页

- 大标题：小说家的新篇章
- 副标题：准备好释放你的创造力了吗？让 AI 引导你，一步步构建出独一无二的故事世界。
- 按钮：开启灵感模式；返回：回创作中心。

### 3. 对话式灵感空间

- 顶部：与「文思」对话中… + 轮次显示；操作：重置 / 关闭（有未沉淀内容时二次确认）。
- 中间：气泡对话流，AI 开场引导（画面 / 一句对白 / 一种感觉）。
- 底部：输入框 + 发送。
- **非流式**，复用 `rendererBridge.generateText()`，含现有取消 / 失败提示 / 60s 超时。
- **轮次上限 8 轮**；达到上限后禁用发送，引导用户生成蓝图（现有 ≥4 轮的软提示保留）。
- 发送给模型的对话历史按轮拼接；超长时从最早轮次截断（保留 system + 开场）。

### 4. 沉淀为创意（关键收口步，2026-07-02 二次修订）

> 修订说明：初版要求「对话 → AI 蒸馏为创意概要 → 预填 idea → 再生成蓝图」。前端实机实现为「对话 → 可编辑蓝图确认页 → 落库时 idea 取用户发言原文拼接」，实质满足「用户确认才写回」约定，且 idea 有落库、可回溯。经复核**采纳实现形态**，不强推 AI 蒸馏步，避免中途重做。以下为采纳后的最终要求：

- 对话 → 生成蓝图（`buildBlueprintFromConversationPrompt`，保留）→ 蓝图进**可编辑草稿**，用户确认后才写回。
- **蓝图确认页须同时展示可编辑的「创意概要」框**（初值 = 用户发言原文拼接），确认时以编辑后的文本落库为 `idea`。零 AI 调用，仅一个输入框。
- `Novel.idea` 落库后须非空（有对话即有发言拼接兜底）。
- 后续大纲 → 章节落库 → 工作台，复用 3a，不改。

## 数据边界（零 schema 改动）

- **对话 session-only，不落盘**：不进 `novel.json`，不加字段，不建对话存储。关闭 / 重置即丢弃（有确认）。
- 唯一持久产物是经用户确认的 `idea`（及其后续 3a 链产物），沿用 3a 的字段与写回边界。
- 不动 IPC、存储、`sanitizeNovel`、主进程任何代码。

## 新增 Prompt

放入 `src/features/novel-creation/novelPrompts.ts`（沿用代码内常量，不做 Registry）：

- `buildInspirationChatPrompt(history)` — 「文思」人设 + 引导式提问，每轮聚焦收敛一个故事要素（题材 / 主角 / 冲突 / 基调），不发散闲聊。
- `buildBlueprintFromConversationPrompt(history)` — 将对话直接生成作品蓝图（结果进可编辑确认页，见第 4 节）。

## 防跑偏约束

3a.1 **不得**实现：

- streaming 输出
- 对话历史持久化 / 多会话管理
- Agent 框架 / 多角色编排
- 7-tab、Bible、多版本、审校、RAG、知识库、Prompt 后台（沿用上位文档禁令）
- 第三张入口卡片或「敬请期待」占位

## 验收标准

1. 小说创作页首页为创作中心两卡片；「小说工作台」进入现有三栏，功能零回归。
2. 灵感模式：启动页 → 对话空间可完整走通；开场引导、逐轮对话、轮次显示、重置、关闭确认均正常。
3. 对话生成支持生成中 / 取消 / 失败提示 / 60s 超时（复用现有机制）。
4. 8 轮上限生效，到顶引导沉淀；第 2 轮起可沉淀。
5. 蓝图确认页含可编辑「创意概要」+ 可编辑蓝图，确认后进入大纲链，最终落成小说项目进入工作台；落库后 `Novel.idea` 为用户确认的文本，非空。
6. 关闭应用 / 刷新后对话不残留；`novel.json` 无新增字段。
7. 不影响续写/润色/改写、生图工作台、资产管理、API 配置。

## 建议实施顺序

1. 确认 3a 验收通过（前置门槛）。
2. 前端：先做创作中心 + 启动页（纯 UI，无 AI 依赖），再做对话空间，最后接沉淀 → 3a 向导。
3. QA：按上述 7 条验收，重点回归第 1、6、7 条。
