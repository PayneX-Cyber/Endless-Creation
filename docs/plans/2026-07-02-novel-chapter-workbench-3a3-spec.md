# 小说创作 3a.3：章节创作工作台 切片规格

日期：2026-07-02
上位文档：`docs/plans/2026-06-30-novel-creation-migration-plan.md`、`docs/plans/2026-07-02-novel-generation-chain-3a-spec.md`、`docs/plans/2026-07-02-novel-inspiration-mode-3a1-spec.md`、`docs/plans/2026-07-02-novel-workbench-project-view-3a2-spec.md`

## 总体结论

3a.3 只做一件事：**把项目卡片「创作」入口背后的现有三栏编辑器视图，重构为章节创作工作台**。它不是新编辑器，不是完整 NovelForge 复刻，不是批量生成系统。

一句话目标：

```text
从「我的小说项目」点击创作 → 进入章节工作台 → 选择章节 → 查看已完成内容或按顺序生成未开始章节
```

入口现状：「创作」按钮已存在且已路由到 `workbench` 视图，3a.3 的实质是**重构该视图**，不新增路由和入口接线。3a.2 项目查看页内的「开始创作」入口也随之进入新工作台。

范围锁定为：**重构 workbench 视图 + 顺序生成首个未完成章节 + 补空 outline**。其它都不碰。

## 一、页面结构

推荐最小布局：

```text
顶部：项目栏
左侧：蓝图与章节列表
右侧：章节创作区
```

**顶部项目栏：**

- 返回（回项目列表）
- 书名
- 副标题摘要：**显示 `novel.summary`（简介）截断**。当前无类型/标签字段，不新增 genre 字段，不发明数据。
- 完成度、已完成章节数 / 总章节数
- 项目详情入口（回 3a.2 查看页）
- **不做菜单占位入口**——当前无真实菜单项，按「不做假入口」原则不展示。

**左侧蓝图 / 章节栏：**

- 故事蓝图标题与摘要（取 `blueprint`，无则回退 `summary` / `idea`，全空显示空态）
- 章节统计：总章节、已完成、未完成
- 章节列表：每章显示标题、outline 摘要、状态标签（已完成 / 生成中 / 未开始）
- 定位第一个未完成章节
- 底部「生成后续大纲」

**右侧章节创作区状态机：**

```text
未选章：选择章节开始创作（空态）
已完成：章节内容展示（标题、outline、正文、字数、状态）
未开始且前序未完成：请先完成前面的章节
第一个未完成章节：显示「按顺序生成」
生成中：loading，禁止重复提交
失败：生成失败，可重试
```

**编辑能力处置：** 右侧正文**保留可编辑**——现有 workbench 的正文 textarea + 自动保存是已落地能力，原样复用，零新增成本；做成只读反而是删功能。现有编辑器上的 AI 续写 / 润色 / 改写按钮**不搬进 3a.3**（后置，不展示）。

## 二、数据口径

**零存储 schema 新增。** 只复用现有字段：

```text
Novel: title, summary, idea, blueprint, chapters
Chapter: title, outline, content, order
```

注意：**`Chapter` 上没有 `summary` 字段**，本切片也不新增。任何"上一章摘要"均以上一章 `content` 尾部摘录替代（见 AI 行为）。

章节状态推导：

```text
content 有值 → 已完成
本会话正在生成 → 生成中
content 为空 → 未开始
```

**「生成中」是会话内瞬时 UI 状态，不持久化**——不写入任何字段，刷新 / 重进后回到未开始。QA 按此口径验收，不按持久化验。

完成度 = 已完成章节数 / 总章节数（复用 `filledChapterCount / chapterCount` 推导链）。

不新增：Bible、版本、关系、角色 schema、RAG 记忆、批量任务表。

## 三、用户流程

首次进入：顶部显示小说信息 → 左侧显示蓝图概要和章节列表 → 右侧空态"选择章节开始创作"。

查看已完成章节：点击 → 右侧显示标题、outline、正文、字数、已完成状态。

生成未开始章节：

```text
点击未开始章节
→ 若前序章节未完成，提示"请先完成前面的章节"
→ 若为第一个未完成章节，显示「按顺序生成」
→ 点击生成 → AI 生成正文草稿
→ 预览 → 用户确认写入 chapter.content
→ 状态变为已完成，完成度更新
```

**顺序限制：只能生成第一个未完成章节。** 这条锁死，省掉批量队列、乱序上下文、复杂冲突处理。

## 四、AI 行为

本切片 AI 面上**只有两处新增**，其余全部复用 3a 已有链路；IPC 优先复用现有 `generateText` 与小说保存接口，不为 3a.3 新增专用通道。

**1. 按顺序生成章节正文（小改，复用为主）**

复用 `buildChapterFromOutlinePrompt` 链路，输入：

```text
novel.title / summary / blueprint
当前章节 title + outline
上一章 content 尾部摘录（如有上一章且有正文）
```

其中"上一章尾部摘录"是**新增的 prompt 改动**——现有 prompt 只带本章自己已写内容的末尾，不带上一章上下文。这是 3a.3 需要如实计入的改动，不是复用。截取长度由实现定，本规格不约定 token 细节。不引入 summary 生成链。

写入规则：不自动覆盖；先预览；确认后写入 `chapter.content`；写入后状态变已完成。沿用现有草稿预览 → 确认写入口径，避免覆盖用户在生成期间的手动编辑。

**2. 生成后续大纲（新增增量变体）**

这不是纯复用：现有大纲向导是"整本生成、整体建章"。3a.3 需要一个**增量 prompt 变体 + 合并逻辑**，单独列为一条实现任务，并锁死边界：

- 输入：idea、blueprint、已有章节的 title / outline / 完成状态
- 输出：缺失章节的 outline 草稿
- 写入规则：预览确认；**只写入 outline 为空的章节；不创建新章节；不覆盖已有 outline**。超出即是大纲系统，不做。

**失败处理（两处通用）：** 保留已有 content 和 outline；显示重试；错误信息脱敏。

## 五、后置清单（不进 3a.3）

- 登录 / 退出登录真实逻辑
- 角色 / 关系真实数据、角色卡、关系图、世界观后台
- 版本管理、查看所有版本、重新生成
- 手动编辑的新流程（现有 textarea 编辑保留，不算新流程）
- 导出 TXT、分层优化
- 批量生成 180 章、自动连续生成多章
- RAG、多版本、Prompt 后台、复杂 Agent
- Bible / schema 扩展、summary 链新能力

**不做假入口：以上一律不展示，不置灰占位。按钮也是债。**

## 六、视觉要求

只取参考图结构，不照搬视觉。延续 Endless Creation 风格：极致清爽、大留白、大圆角、无硬边框、微阴影、蓝紫主题；顶栏轻量；左栏像创作导航，右侧像沉浸写作台。避免：密集表格、重边框、过多按钮、大量彩色标签、后台感。

## 七、验收标准

入口：

1. 「我的小说项目」卡片点击「创作」进入 3a.3 工作台；「查看」仍进 3a.2；顶部返回可回项目列表。
2. 3a.2 内「开始创作」进入 3a.3 工作台；若调用方提供 chapterId，则定位对应章节，否则显示未选章空态；不破坏查看页返回链路。

顶部栏：

3. 显示书名、`summary` 截断、完成度、已完成 / 总章节数；项目详情可回 3a.2。

左侧栏：

4. 显示蓝图摘要（blueprint → summary → idea 回退）、章节统计、章节列表（标题 / outline 摘要 / 状态）。
5. 可定位第一个未完成章节；可触发生成后续大纲。

右侧区：

6. 未选章显示空态；点击已完成章节显示正文和字数，且正文可编辑并自动保存。
7. 点击未开始章节：前序未完成时提示先完成前面章节；第一个未完成章节显示「按顺序生成」。
8. 生成中有 loading、禁止重复提交；**生成中状态不持久化**，刷新后回到未开始。
9. 生成失败可重试，已有内容不丢。
10. 生成结果先预览、确认后写入；写入后状态变已完成、完成度更新。
11. 生成后续大纲只补齐空 outline 章节，不创建新章节，不覆盖已有 outline，预览确认后写入。

边界：

12. 零存储 schema 新增（无 chapter.summary、无 Bible、无版本）；不新增 AI Provider / RAG / 多版本；IPC 优先复用现有接口，不为 3a.3 新增专用通道。
13. 不做真实登录退出、导出 TXT、分层优化、重新生成、批量生成。
14. 不破坏小说 CRUD、3a 生成链、3a.1 灵感模式、3a.2 查看页、生图与资产模块。
