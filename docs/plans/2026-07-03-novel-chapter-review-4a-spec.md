# 小说创作 4a：章节评审 切片规格

日期：2026-07-03
上位文档：`docs/plans/2026-06-30-novel-creation-migration-plan.md`（第四阶段「评估与优化」）、`docs/plans/2026-07-03-novel-multi-version-3b-spec.md`

## 总体结论

4a 只做一件事：**给已完成章节加一个只读的「AI 评审」**。它不是六维审稿系统，不是一致性检查器，不是优化改写工具。

一句话目标：

```text
已完成章节 → 点击「章节评审」 → AI 基于蓝图/大纲/正文给出一段评审意见 → 只读展示，随手关闭
```

这是 6-30 总纲第四阶段的第一刀。第四阶段的本质是「Prompt + 当前小说上下文 + 文本生成接口」，4a 用最小闭环先证明这条评估链路；节奏检查、人物一致性、对话/环境/心理优化全部后置。

范围锁定为：**一个评审 prompt + 一个只读评审面板**。其它都不碰。

## 一、数据口径

**零 schema 新增，零落库。**

- 评审结果是**会话内瞬时数据**：只存组件 state，不写 `novel.json` 任何字段，不进 localStorage，刷新/切章即丢。
- 不新增：评分字段、评审历史、六维结构、consistency 数据、Bible、summary 链。
- 这是 4a 最锁死的边界——先验证「评审意见是否有用」，再决定要不要为它建结构（那是后续切片的事）。

## 二、用户流程

入口（3a.3 章节工作台，不新增页面）：

```text
已完成章节的编辑器 meta 行 → 「章节评审」按钮（与「历史版本」同级的轻量入口）
→ 点击 → 评审中（loading，禁止重复提交，可取消）
→ 完成 → 弹出只读评审面板（复用现有 modal 风格）
→ 关闭即丢，可重新评审
```

边界：

- 只对**已完成章节**（content 非空）提供入口；未开始/生成中章节不显示，不置灰占位。
- 评审期间可正常关闭面板/切章，行为与现有生成取消一致。
- 失败：显示重试、错误脱敏，不影响正文与版本数据（本来也不写）。

## 三、AI 行为

- 新增一个 prompt：`buildChapterReviewPrompt(novel, chapter)`，放 `novelPrompts.ts` 代码内常量（沿用既有方式，不做 Prompt Registry）。参考迁移源 `evaluation.md` 的意图，但**输出为一段自由文本评审**（优点 / 问题 / 修改建议），不要求结构化评分。
- 输入：`novel.title / summary / blueprint`、本章 `title / outline / content`（超长正文按尾部截断，长度由实现定）。
- 复用现有 `generateText` IPC 与模型偏好，零新增通道；生成中 / 取消 / 60s 超时 / 错误脱敏全部复用。
- 不做：多章联合评审、上一章上下文拼接、自动按建议改写。

## 四、视觉要求

延续工作台风格：评审面板复用「历史版本」同款轻量 modal；评审文本按段落展示；一个「重新评审」ghost 按钮 + 关闭。不做评分卡、雷达图、批注定位。

## 五、后置清单（不进 4a）

- 六维审稿、结构化评分、评审历史落库
- 人物一致性检查、轻量 consistency checker（第二参考源登记项）
- 对话 / 环境 / 心理 / 节奏优化及其写回流程（涉及 diff/写回 UX，独立切片）
- 按评审建议一键改写、读者模拟、自我批判
- Prompt Registry、任务模型路由

**不做假入口：以上一律不展示，不置灰占位。**

## 六、验收标准

1. 已完成章节 meta 行显示「章节评审」入口；未开始 / 生成中章节不显示。
2. 点击后进入评审中状态：loading、禁止重复提交、可取消；与正文生成/大纲生成互斥（同一时刻只有一个 AI 任务）。
3. 评审完成弹出只读面板，展示评审文本；可「重新评审」；关闭即丢。
4. 评审失败显示重试，错误信息脱敏；正文、大纲、版本数据不受影响。
5. **零落库**：评审前后 `novel.json` 内容不变（不新增字段、不触发保存）。
6. 不破坏 3a/3b 全链路（顺序生成、多版本、大纲补齐、灵感模式、查看页）、小说 CRUD、生图与资产模块。
7. 零新增 IPC / Provider / schema；仅新增一个 prompt 函数与工作台内 UI。

## 建议实施顺序

1. 前端：prompt 函数 → 入口按钮与互斥 → 评审面板。
2. QA：验收 7 条 + 回归 3a/3b 关键路径（防覆盖、多版本写入）。
