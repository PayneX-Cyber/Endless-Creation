# v0.3 Electron 最小壳 QA 验收清单

适用范围：Endless Creation v0.3 Electron + React 桌面端最小壳。  
目标：验证 Electron main/preload/renderer 最小链路可运行，同时保证 Web fallback、主题和 mock 生成交互不退化。

## 0. 前置条件

- 在仓库根目录执行命令。
- 已安装现有 `package-lock.json` 对应依赖。
- 不引入大型测试框架；本清单以可复测手动验收 + 轻量命令检查为主。
- 测试时打开 DevTools Console 与运行终端，记录错误输出。

## 1. 命令与构建

### TC-01：`npm run build`

步骤：

```powershell
npm run build
```

预期结果：

- `build:renderer` 通过：TypeScript build 与 Vite build 成功。
- `build:electron` 通过：Electron main/preload TypeScript 编译成功。
- 生成 `dist/` 与 `dist-electron/`。
- 终端无 TypeScript error、Vite error、Electron 编译 error。

### TC-02：`npm run check`

步骤：

```powershell
npm run check
```

预期结果：

- 作为轻量 QA 门禁执行成功。
- 当前最小实现中应等价覆盖 `npm run build`。
- 失败时必须阻塞合并。

### TC-03：`npm run dev:electron`

步骤：

```powershell
npm run dev:electron
```

预期结果：

- Vite renderer dev server 在固定端口启动。
- Electron main 编译后启动。
- 出现桌面窗口。
- 终端无 main process crash。
- 关闭窗口后进程能正常退出或由 dev runner 清理。

## 2. Electron 窗口启动

### TC-04：默认窗口可见

步骤：

1. 执行 `npm run dev:electron`。
2. 等待窗口出现。

预期结果：

- 窗口标题为 `Endless Creation` 或产品约定标题。
- 默认窗口尺寸足以展示主要工作台。
- 首屏无白屏、无透明异常、无内容被裁切。
- 背景色与深色主题一致，无加载闪烁到错误页面。

### TC-05：窗口尺寸边界

步骤：

1. 尝试缩小窗口到最小尺寸。
2. 尝试最大化、还原。
3. 在约 960、1024、1280、1440 宽度下观察布局。

预期结果：

- 最小窗口尺寸限制生效。
- 最大化 / 还原无崩溃。
- renderer 布局不出现整体水平溢出。
- Prompt、结果区、导航和主题按钮仍可操作。

## 3. Renderer 正常加载

### TC-06：Electron 内加载 React renderer

步骤：

1. 启动 `npm run dev:electron`。
2. 查看 Electron 窗口和 DevTools Console。

预期结果：

- Electron 窗口显示 React 工作台。
- 能看到导航、Prompt、生成按钮、最近项目、结果区域。
- Console 无 `Failed to load URL`、`ERR_FILE_NOT_FOUND`、React runtime error。
- 刷新窗口后 renderer 仍能恢复。

### TC-07：生产构建 renderer 加载路径

步骤：

1. 执行 `npm run build`。
2. 确认 `dist/index.html` 与 `dist-electron/main/index.js` 存在。
3. 如有 packaged / production 启动命令，使用该命令启动；若暂无，只记录为后续打包验证项。

预期结果：

- Electron main 在无 `VITE_DEV_SERVER_URL` 时应加载 `dist/index.html`。
- 不依赖 dev server 才能加载 renderer。

## 4. Preload bridge

### TC-08：bridge 对象可用

步骤：

在 Electron DevTools Console 执行：

```js
typeof window.endlessCreationBridge
Object.keys(window.endlessCreationBridge || {})
```

预期结果：

- `window.endlessCreationBridge` 为对象。
- 只包含白名单域，例如 `app`、`window`、`clipboard`。
- 不暴露 `ipcRenderer`、`require`、`fs`、`path`、`child_process` 等底层能力。

### TC-09：app bridge

步骤：

```js
await window.endlessCreationBridge.app.getVersion()
await window.endlessCreationBridge.app.getPlatform()
```

预期结果：

- 返回字符串。
- Promise 正常 resolve。
- Console 无 unhandled rejection。

### TC-10：window bridge

步骤：

1. 点击 UI 窗口控制按钮，或在 Console 调用：

```js
await window.endlessCreationBridge.window.minimize()
```

2. 手动恢复窗口。
3. 调用 maximize / close 前确认当前测试允许关闭窗口。

预期结果：

- minimize / maximize / close 只影响当前窗口。
- 调用失败时返回可处理错误，不导致 renderer 崩溃。

### TC-11：clipboard bridge

步骤：

1. 通过 UI 生成成功结果。
2. 点击 `复制结果`。
3. 粘贴到文本编辑器验证内容。

预期结果：

- Electron 环境优先走 preload clipboard bridge。
- 显示 `已复制`。
- 剪贴板内容等于生成结果文本。
- bridge 不可用时 Web fallback 应显示可理解失败，不崩溃。

## 5. 安全配置

### TC-12：renderer 不暴露 Node

步骤：

在 Electron DevTools Console 执行：

```js
typeof require
typeof process
typeof Buffer
window.require
```

预期结果：

- `require` 不可用。
- `window.require` 为 `undefined`。
- 不可直接访问 `fs`、`child_process`。
- 若 `process` 因 Electron 环境部分存在，也不得允许通过它获得 Node 文件/进程能力。

### TC-13：BrowserWindow 安全配置审查

检查 `electron/main/index.ts`：

- `contextIsolation: true`
- `nodeIntegration: false`
- `preload` 指向固定 preload 编译文件
- 不关闭 `webSecurity`
- 不启用 remote module
- 不允许 renderer 任意 IPC channel 调用

预期结果：

- 本地能力只能通过 preload 白名单 API 访问。
- IPC handler 对参数做类型校验，例如 clipboard text 必须为 string。

### TC-14：IPC 边界

步骤：

1. 调用 bridge 的公开方法。
2. 尝试查找是否可从 renderer 直接访问 `ipcRenderer`。

预期结果：

- 只能调用 bridge 封装方法。
- 无任意 channel invoke 能力暴露给 renderer。
- 错误参数不会导致 main 崩溃。

步骤：

```powershell
npm run dev -- --host 127.0.0.1 --port 5173
```

打开：

```text
http://127.0.0.1:5173/
```

预期结果：

- 浏览器中可正常显示工作台。
- 不要求 `window.endlessCreationBridge` 存在。
- 无 Electron-only API undefined 崩溃。
- 主题、Prompt 校验、mock 生成、复制 fallback 仍可用或显示合理失败。

### TC-16：Web build preview

步骤：

```powershell
npm run build
npm run preview
```

预期结果：

- Web 生产预览可打开。
- 不依赖 Electron bridge。
- Console 无 runtime error。

## 7. 主题与生成交互不退化

### TC-17：主题切换

步骤：

1. 在 Electron 中点击主题切换。
2. 刷新 renderer。
3. 在 Web fallback 中重复测试。

预期结果：

- 深色 / 浅色即时切换。
- `ec-theme` 持久化可用。
- 新增 Electron bridge 不影响 Web localStorage fallback。

### TC-18：active 导航

步骤：

1. 依次点击 `文本生成`、`图片生成`、`视频生成`、`项目库`。
2. 使用键盘 Tab + Enter/Space 重复。

预期结果：

- active 状态唯一。
- `aria-current` 只出现在当前项。
- 当前模式影响生成结果标题和内容。

### TC-19：Prompt 校验

步骤：

1. 清空 Prompt。
2. 输入纯空格。
3. 点击生成。

预期结果：

- 显示校验错误。
- 不进入 loading。
- 不创建 mock task。
- 输入有效内容后错误自动清除。

### TC-20：mock 生成 loading / success / error / retry

步骤：

1. 输入有效 Prompt。
2. 点击生成。
3. loading 期间重复点击。
4. 多次生成直到覆盖 success 与 error。
5. error 时点击重试。

预期结果：

- loading 时按钮 disabled。
- success 显示结果摘要、正文和复制按钮。
- error 显示错误文案和重试按钮。
- 重试使用当前 Prompt 和当前模式。
- Console 无 unhandled rejection。

## 8. 最小通过标准

v0.3 最小壳可验收条件：

- `npm run build` 通过。
- `npm run check` 通过。
- `npm run dev:electron` 可启动窗口。
- Electron 窗口正常加载 renderer。
- Preload bridge 可访问且只暴露白名单 API。
- Renderer 不能直接访问 Node 原始能力。
- Web fallback `npm run dev` 仍可运行。
- 主题与生成交互在 Electron / Web 双模式不退化。
- 主流程无 Console error、无 main process crash。
